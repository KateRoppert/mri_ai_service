## 1. Problem Statement

The current skull stripping in Stage 05 uses FSL BET (`-f 0.35 -g -0.1 -R`), selected empirically.
For heterogeneous clinical data (GBM + MS, multiple acquisition centers, varying protocols)
BET may not be optimal. This research branch:

1. Systematically compares skull stripping tools across lesion types and datasets
2. Establishes an evidence base for tool selection (for publication)
3. Introduces a plugin architecture in Stage 05 that makes the tool config-driven and MAS-ready
4. Identifies 2 production winners to be deployed as separate Docker agents in Этап 7

## 3. Datasets

### Benchmark datasets

| Dataset | Type | Access | Role in paper |
|---|---|---|---|
| **UPENN-GBM** | Glioblastoma | Open (local: `data/UPENN-GBM`) | Primary open GBM results |
| **MosMed Sclerosis** | Multiple Sclerosis | Open (download required, verify license) | Primary open MS results |
| **Local GBM** | Glioblastoma | Closed (clinical) | Full analysis; reported as "proprietary clinical dataset" |
| **Local MS** | Multiple Sclerosis | Closed (P000915 + others) | Full analysis; reported as "proprietary clinical dataset" |

Closed datasets are included in full analysis (not only validation) because they represent
the target clinical distribution (same acquisition centers as production deployment).
Published metrics only — raw data not shared.

### Subject selection
- Stratified sampling: 20–30 subjects per dataset for parameter tuning + benchmark
- Split: **20% validation** (parameter tuning), **80% test** (final evaluation, single run)
- Split is at subject level, not slice level

### Input data format
All benchmark inputs are NIfTI images **after registration to MNI152 atlas, before skull stripping**.
Generated by `prepare_data.py` which runs pipeline stages 1–4 (reorient → bias correction →
registration) with `skull_stripping: disabled`.
Note: bias correction runs with `use_for_registration_only: true` — registration uses
bias-corrected image but output saved is the original (non-corrected). `prepare_data.py`
must save the registration-space image explicitly as benchmark input.

**Atlas standardization:** MNI152 for all datasets. GBM benchmark overrides SRI24 config
with MNI152 for consistency. This is benchmark-only; production configs unchanged.
MNI152 template: `data/templates/MNI152_T1_1mm.nii.gz` (already in repo).

## 4. Metrics

### Quality metrics (per subject × tool × dataset)

| Metric | Formula | Notes |
|---|---|---|
| **DSC** | `2|A∩B| / (|A|+|B|)` | vs MNI152 brain mask as pseudo-GT |
| **HD95** | 95th percentile Hausdorff distance | Boundary accuracy |
| **Over-stripping rate** | `FN / (TP+FN)` | How much brain is cut |
| **Leakage rate** | `FP / (TN+FP)` | Non-brain tissue in mask |
| **Brain volume (ml)** | mask voxels × voxel size | Sanity check |

**Limitation note (for paper):** Atlas mask as pseudo-GT is imperfect for GBM cases with
mass effect — registration is not perfect for distorted anatomy. Supplement with
qualitative visual assessment (5–10 subjects per dataset, 1–3 scale, assessed by KR).

### Performance metrics (per tool × hardware)

| Metric | Method |
|---|---|
| **Processing time (s/volume)** | `time.perf_counter()` |
| **RAM peak (GB)** | `tracemalloc` + `psutil` |
| **VRAM peak (GB)** | `pynvml` before/after |
| **Reproducibility** | DSC between two identical runs on same subject |

**Hardware:** NVIDIA RTX 5070 12GB VRAM, documented in paper Methods section.

## 5. Statistical Analysis

### Test hierarchy

```
1. Shapiro-Wilk normality test (n<50) or D'Agostino (n≥50)
2. If normal  → two-way ANOVA (factors: tool × dataset)
   If not     → Kruskal-Wallis + pairwise Wilcoxon signed-rank
3. Multiple comparisons correction → Bonferroni (7 tools = 21 pairs)
4. Effect size → η² (ANOVA) or rank-biserial r (Wilcoxon)
5. Overall tool ranking → Friedman test (same subjects, all tools)
```

### Research questions answered

| Question | Test |
|---|---|
| Which tool is best overall? | Friedman + post-hoc Wilcoxon |
| Is there tool × lesion_type interaction? | Two-way ANOVA interaction term |
| Is any GPU tool worth the cost vs atlas mask? | Effect size: HD-BET vs MNI-loose |
| Does tuning help significantly vs defaults? | Paired t-test: default vs tuned DSC |

### Python libraries
`scipy`, `pingouin`, `statsmodels`, `pandas`, `matplotlib`, `seaborn`
Added to `research/requirements-research.txt`.

## 6. Parameter Tuning

Targeted search on **key parameters only** (based on literature recommendations):

| Tool | Parameter | Search range |
|---|---|---|
| FSL BET | `fractional_intensity` | 0.2, 0.3, 0.35, 0.4, 0.5 |
| HD-BET | `threshold` | 0.3, 0.5, 0.7 |
| SynthStrip | `--border` | 0, 1, 2 mm |
| MNI masks | dilation radius | fixed (strict=0, loose=2mm) |

Tuning runs on **validation split only**. Best params per tool saved to `manifest.yaml`.
Final evaluation uses tuned params on test split, single run.

## 8. Plugin Architecture

### SkullStripperBase contract

```python
class SkullStripperBase(ABC):
    @abstractmethod
    def strip(self, input_path: Path, output_path: Path,
              mask_path: Path, params: dict) -> dict:
        # returns: {success, mask_path, processing_time, vram_used_gb, error?}
        ...

    @abstractmethod
    def is_available(self) -> bool: ...

    @property
    @abstractmethod
    def manifest(self) -> dict: ...   # loaded from manifest.yaml
```

### Dispatcher behavior

1. Reads `method:` from `preprocessing_config.yaml`
2. Calls `is_available()` on selected stripper
3. If unavailable → switches to `fallback_method:` with WARNING in log
4. Returns stripper instance; `process_subject_skull_stripping` signature unchanged

### Config extension

```yaml
- name: skull_stripping
  enabled: true
  params:
    method: "hdbet"           # bet | hdbet | synthstrip | mni_mask | ...
    reference_modality: "t1"
    apply_to_all: true
    cleanup: true
    fallback_method: "bet"    # used if primary is unavailable
    tool_params:              # tool-specific, from manifest tuning
      threshold: 0.5
```

### Manifest structure

```yaml
# services/skull-stripping/hdbet/manifest.yaml
name: hdbet
version: "1.0"
tool_type: skull_stripping
compute:
  requires_gpu: true
  gpu_memory_gb: 4
  ram_gb: 8
  cpu_fallback: true
lesion_type_preference:
  glioma: high
  ms: high
  metastases: medium
modality_support:
  primary: [t1, t1c]
  apply_to: [t1, t1c, t2, t2fl]
quality_notes: >
  Best current choice for tumor multi-modal MRI (2019).
  Validated on BraTS-like data.
install:
  method: pip
  package: hd-bet
  model_download: auto
mas_metadata:
  agent_type: skull_stripper
  cascade_priority: 1
  fallback_to: synthstrip
```
